# [바이러스](https://github.com/malvr00/Java-algorithm/blob/master/backjoon/dfs_bfs/step3/src/Main.java)
신종 바이러스인 웜 바이러스는 네트워크를 통해 전파된다. 한 컴퓨터가 웜 바이러스에 걸리면 그 컴퓨터와 네트워크 상에서 연결되어 있는 모든 컴퓨터는 웜 바이러스에 걸리게 된다.<br/>
<br/>
예를 들어 7대의 컴퓨터가 <그림 1>과 같이 네트워크 상에서 연결되어 있다고 하자. 1번 컴퓨터가 웜 바이러스에 걸리면 웜 바이러스는 2번과 5번 컴퓨터를 거쳐 3번과 6번 컴퓨터까지 전파되어 2, 3, 5, 6 네 대의 컴퓨터는 웜 바이러스에 걸리게 된다. 하지만 4번과 7번 컴퓨터는 1번 컴퓨터와 네트워크상에서 연결되어 있지 않기 때문에 영향을 받지 않는다.<br/>
<br/>
어느 날 1번 컴퓨터가 웜 바이러스에 걸렸다. 컴퓨터의 수와 네트워크 상에서 서로 연결되어 있는 정보가 주어질 때, 1번 컴퓨터를 통해 웜 바이러스에 걸리게 되는 컴퓨터의 수를 출력하는 프로그램을 작성하시오.

([자세한 문제는 백준 참고](https://www.acmicpc.net/problem/2606)) <br/>

## 해법
* 양방향 그래프 문제이기 때문에 양방향으로 `node`의 값을들 배열에 저장한다.
* 중복 탐색을 방지하기위해 방문한 `node`의 위치 값을 확인하는 확인용 체크배열을 하나 만든다.
* 준비가 끝나면 재귀호출을 통해 그래프를 탐색한다.
  * 함수에서 `root` 해당 바이러스 걸린 지점이 도착하면 해당 위치는 체크배열로 바이러스가 감염됐다는 표시 또는 방문을 했기 때문에 해당 위치에 값을 `TRUE`로 변경한다.
  * 해당 `root`와 연결된 `node`가 방문하지 않았던 `node`라면 `감염 카운트를 1 증가`시킨다. 모든 `node` 재귀호출하면서 연결된 부분을 모두 탐색한다.

## 핵심
* 양방향으로 연결되어있기 때문에 이 부분을 생각해서 만들면 쉽게 문제를 풀 수 있다.
* 바이러스 감염은 한번 만 이러워지면 되기 때문에 바이러스 감염된 `node`는 재방문 하지 않기 위해 방문했다는 체크배열을 만들면 쉽게해결가능하다.

## 참고
