# [줄세우기](https://github.com/malvr00/Java-algorithm/blob/master/backjoon/dp/step25/src/Main.java)
어떤 극장의 좌석은 한 줄로 되어 있으며 왼쪽부터 차례대로 1번부터 N번까지 번호가 매겨져 있다. 공연을 보러 온 사람들은 자기의 입장권에 표시되어 있는 좌석에 앉아야 한다. 예를 들어서, 입장권에 5번이 쓰여 있으면 5번 좌석에 앉아야 한다. 단, 자기의 바로 왼쪽 좌석 또는 바로 오른쪽 좌석으로는 자리를 옮길 수 있다. 예를 들어서, 7번 입장권을 가진 사람은 7번 좌석은 물론이고, 6번 좌석이나 8번 좌석에도 앉을 수 있다. 그러나 5번 좌석이나 9번 좌석에는 앉을 수 없다.<br/>
<br/>
그런데 이 극장에는 “VIP 회원”들이 있다. 이 사람들은 반드시 자기 좌석에만 앉아야 하며 옆 좌석으로 자리를 옮길 수 없다.<br/>
<br/>
오늘 공연은 입장권이 매진되어 1번 좌석부터 N번 좌석까지 모든 좌석이 다 팔렸다. VIP 회원들의 좌석 번호들이 주어졌을 때, 사람들이 좌석에 앉는 서로 다른 방법의 가짓수를 구하는 프로그램을 작성하시오.<br/>
<br/>
예를 들어서, 그림과 같이 좌석이 9개이고, 4번 좌석과 7번 좌석이 VIP석인 경우에 <123456789>는 물론 가능한 배치이다. 또한 <213465789> 와 <132465798> 도 가능한 배치이다. 그러나 <312456789> 와 <123546789> 는 허용되지 않는 배치 방법이다.

([자세한 문제는 백준 참고](https://www.acmicpc.net/problem/2302)) <br/>

## 해법 1
* 죄석 번호만큼 배열 `dp`를 만든다.
* 주어지는 `VIP` 좌석 번호를 `VIP` 좌석인지 확인하기 위해 `Set`에 저장해서 탐색할 때 활용한다.
* `i = i 위치 까지의 앉는 방법`으로 반복문을 돌면서 `i` 위치에 최대 앉을 수 있는 방법 가짓수를 저장해야한다.
* 점화식은 아래와 같다.
  * `if(Set.contains(i) || Set.contains(i - 1) dp[i] = dp[i - 1]`
    * `i`의 위치와 `i` 옆 자리가 `VIP`인 경우 좌석이동이 발생하지 않기 때문에 이전에 구한 가짓수를 계승한다.
  * `else dp[i] = dp[i - 2] + dp[i - 1]`
    * `i`의 위치에 좌석이동이 자유롭다면 인접한 사람끼리 좌석 이동한 가짓수 누적합 진행한다.

## 해법 2
* 죄석 번호만큼 배열 `dp`를 만든다.
* `i = (VIP를 생각하지않고) i 위치 까지의 앉는 누적합` 반복문을 돌면서 `ith` 까지 앉을 수 있는 가짓 수가 각각 저장되어야한다.
* 맨 처음 `VIP`는 상관하지 않고 점화식을 활용해 좌석 개수 별 가짓수를 저장한다.
* 점화식은 `dp[i] = dp[i - 2] + dp[i - 1]`와 같다.
* 배열에 앉을 수 있는 가짓수를 전부 저장했으면 `VIP` 좌석을 활용해서 최대 앉을 수 있는 가짓수를 찾는다.
  * `VIP`의 좌석이 존재한다면 `VIP 좌석위치 - 탐색 시작위치 - 1`의 이동이 자유로운 좌석 개수의 최대 가짓수를 정답 변수에 곱해준다.
    * 예를 들어서 `VIP 좌석 위치 = 4, 탐색 시작위치 = 0, 정답변수 = 1` 일 때 자유로운 이동이 가능한 좌석은 `3`이 나오고 자유롭게 이동 가능한 좌석 `3`의 최대 가짓수를 곱해준다. 그러면 `정답변수 = 3`을 가진다.
    * 그 다음 `VIP 좌석 위치 = 7, 탐색 시작위치 = 4 (이전 VIP 위치), 정답변수 = 3` 일 때 자유로운 이동이 가능한 좌석은 `4와 7` 사이인 `{5, 6}`으로 `2`가 나오고 `2`의 최대 가짓수를 정답변수에 곱해주면 `6`을 가진다.
    * 마지막으로 `VIP 좌석 위치 = NONE, 탐색 시작위치 = 7 (이전 VIP 위치), 정답변수 = 6` 일 때 좌석의 크기가 `9`라면 아직 앉을 수 있는 좌석이 `{8, 9}`로 좌석 두 좌석이 남아있고 남은 좌석의 최대 가짓수를 위와 같이 곱해주면 `12`를 가진다.
      * 곱해주는 이유는 `({1,2,3}, {5,6})` 최대 앉을 수 있는 좌석 가짓수는 대칭으로 존재하기에 곱해줘야한다.

## 핵심
* `VIP`를 어떻게 처리할지를 고민해봐야한다.
* 배열의 인덱스 정의를 잘 해야 쉽게 접근할 수 있다.
