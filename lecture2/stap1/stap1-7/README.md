# [비밀번호](https://github.com/malvr00/Java-algorithm/blob/master/lecture2/stap1/stap1-7/src/Main.java)

랜덤으로 주어지는 숫자배치 키패드에서 비밀번호의 값이 들어올때 비밀번호를 누르고있는 위치를 기준으로 8 방향을 탐색했을 때 다음 누를 비밀번호가 존재한다면 +1, 방향 범위안에 존재하지 않는다면 +2, 누르고있는 위치와 같다면 +0을 누적해 누적 값을 출력하는 문제이다.<br/>

## 내가 푼 방식
* 첫 번째: 먼저 키패드 ‘[3][3]’ 크기로 새로 할당해 준다.
* 두 번째; 비밀번호를 누르고있는 번호와 다음 눌러야 할 번호 값을 preVal과 postVal에 저장한다.
* 세 번째: preVal을 기준으로 8 방향을 탐색 후 postVal의 값이 존재한다면 +1, 존재하지 않는다면 +2, preVal와 값이 같다면 +0을 해준다.

<br/>

## 강의
* 첫 번째: 키패드 할당한다.
* 두 번째: dist라는 범위탐색하는 배열을 만들어준다.
* 세 번째: dist를 모두 2로 초기화해 준다.
* 네 번째: dist에서 dist[i][i] 값을 모두 0으로 초기화해 준다. ( i의 i 값은 자기 자신이기 때문에 0으로 초기화 )
* 다섯 번째: 키패드 0, 0 부터 8 방향 범위검색을 통해 dist의 값을 1로 초기화해 준다. ( 키패드[0][0]의 값을 dist[캐패드[0][0]] 로 활용한다. )
* 여섯 번째: 비밀번호를 기준으로 dist를 탐색해 몇 초 걸리는지 값을 누적시킨다.

<br/>

## 차이점
내 문제풀이 방법은 키패드를 기준으로 패스워드 값을 하나하나 탐색하는 반면 강의에서는 dist에 미리 값을 세팅해 그 값으로 정답을 출력하는 방식이다.  <br/>
비밀번호 최대 길이인 20만을 기준으로 탐색을 한다면, 내 문제 풀이 방법은 20만^8 으로 탐색하는데 강의에서는  3^3으로 탐색 후 20만을 O(n)으로 탐색한다. 내 문제풀이 방법으로 한다면 테스트케이스가 20만에 가까워진다면 시간초과가 날 가능성이 높다.
